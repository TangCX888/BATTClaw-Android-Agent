# 角色分配说明

## [执行者](./executor.ts)
**文件：** ` ./executor.ts ` 

**说明：** 用来执行单步的任务，例如点击，输入等，内置循环步数，默认5-8步循环。



# 流程分配
<div class="processTable">

|步骤|角色|执行说明|备注|
|-----|-------|-----------|------------------------|
|1|用户`user`|用户发布任务|例如用户发布任务：` 帮我对比下淘宝和京东上回力男鞋的价格 `|
|2|计划者`planner`|思考决策|对用户发布的任务思考，决策并拆分，例如将用户的任务拆分为：`1. 打开淘宝 2. 搜索回力男鞋 3. 统计前20条数据的价格 4. 打开京东 5. 搜索回力男鞋 6. 统计20条数据的价格` 并挨个递交给`执行者`|
|3|执行者`executor`|执行任务|从`计划者`拆分出的小任务中执行任务并返回结果给`观察者`|
|2|观察者`observer`|检查任务|`执行者`执行任务如果申请完成任务、任务超时、任务失败等情况时会移交给`观察者`，观察者来检查任务是否成功完成，如果成功完成则统计历史记录返回给`计划者`，如果未完成则分析原因并`决策是否让执行者重新执行还是返回给计划者`|

</div>
