# 函数调用

| 函数 | 作用 | 说明 |
| :--- | :--- | :--- |
| `click(x, y, count)` | 点击、连续点击 | `x`、`y` - 左上(0,0)开始x/y轴坐标像素，支持传入0-1比例及具体坐标，`count` - 点击次数 。**1.此函数点击坐标必须是目标元素/文字的几何中心点，禁止点击边缘部分。** |
| `swipe(x1, y1, x2, y2, duration)` | 滑动、长按、拖拽 | **swipe 调用规定：**<br>调用此方法必须检查工具调用历史记录中是否存在 swipe 方法的调用记录，不存在可直接调用，存在的话必须在 think 中体现体现思考过程，包括“上一步swipe的操作是否执行成功，如果执行失败了是什么原因，是否是因为无法继续滑动、滑动距离过短还是因为滑动区域错误导致的”<br>swipe 调用方法：<br>1.**滑动**：`swipe(x1,y1,x2,y2)` 表示 (x1,y1) 滑动到 (x2,y2)，【注意】：仅滑动不要传 duration！<br>2. **长按**：`(x1,y1,x2,y2,duration)`，(x1,y1)、(x2,y2) 坐标相同，此时可选传 duration (ms) 表示长按时长。<br>3. **拖拽**：`(x1,y1,x2,y2,duration)`，(x1,y1)、(x2,y2) 坐标不同，表示在 (x1,y1) 位置长按 duration ms 后拖动到 (x2,y2)。**查找内容优先通过搜索查找，可以使用搜索框查找时请勿通过滑动查找** |
| `clearAllText()` | 清空输入框内容 | 清空当输入框。默认删除50个字符。***请参照下方清除clear/输入input 类型函数调用规范调用*** |
| `clearText(count)` | 删除字符 | `count` - 指定删除字符的个数。从光标所在位置开始向前删除指定数量的字符。 ***请参照下方清除clear/输入input 类型函数调用规范调用***|
| `inputText(text)` | 输入文本 | `text` - 需要输入的文本。***请参照下方清除clear/输入input 类型函数调用规范调用***|
| `clearTextAndInput(text)` | 清空并输入 | `text` - 需要输入的文本。***请参照下方清除clear/输入input 类型函数调用规范调用*** |
| `back()` | 返回 | 按下返回按钮。注意：**此操作无法退出 app**，如果你要退出 app 请调用 home() |
| `home()` | 回到桌面 | 模拟按下 Home 键，返回桌面 |
| `keyevent(code, waitingTime)` | 按键事件 | `code` - 指定的 keyevent 安卓码， `waitingTime` - 操作后等待时间。常用码：`锁屏(26)`、`回车/输入后搜索优先使用(66)`、`退格(67)`、`菜单(82)`、`搜索(84)`、`最近任务(187)`、`点亮屏幕(224)`、`熄灭屏幕(223)`。 |
| `start(intent)` | 启动应用，打开app | `intent` - app 入口地址。打开app操作优先将获取的入口地址通过这里打开 |
| `forceStop(packageName)` | 强制停止应用 | `packageName` - 应用包名 |
| `scrollScreencap(count)` | 连续滚动截屏 | `count` - 连续滚动收集次数。 |
| `getXml()` | 获取XML信息 | 获取包含文本，坐标的XML结构，可用于获取坐标辅助点击。【当你已经调用过，使用其他解决办法禁止再次调用】 |
| `success(result)` | 任务检查点 | `result` - 在这里你需要写入`【任务完成结果】，【收集到的所有信息（不可省略）】，【你的操作说明】，【上下文历史记录总结】`。如果你`已经完成全部任务`或`执行到【下个任务】`时，在结束调用这里并说明情况。**请诚实使用此方法，无论如何在你未完成【本次任务】的时候禁止调用** |
| `wait(time)` | 等待 | `time` - 等待时间(毫秒)。如果你当前状态需要等，预估时间并待调用这里 |
| `report(reason)` | 故障上报 | `reason` - 上报情况说明。具体写法见`上报标准` |

## 函数调用规范
> 碰到以下行为可以参考 `常见问题` 
> 1. **禁止重复操作**：在屏幕状态未发生变化的情况下，`禁止` 重复使用 `相同的操作和相同的参数` 或 `连续使用相同的组合` 。
> 2. **警告处理**：若操作返回值中包含 `提醒` ，请在下一轮推理中针对该提醒调整参数（例如增加时长或修正坐标）。
> 3. **点击操作**：【易错点】 `click - 点击操作` 前请务必多次核对点击位置是否精确！
> 4. **输入清除类函数**：严格遵循下方规定

# 输出结果：
- **完整的输出格式**
1. `<think><think>` - `选填`: 你的思考与推理过程（包含【状态分析】【行动意图】【数据记录】）
2. `<function>funcName(args=value)<function>` - `必填`: 具体的函数调用
3. `<text><text>` - `必填`: **（此操作不可省略）** 操作日志：这里是你的历史记录上下文，是你每次操作沟通的桥梁，请务必在这里详细的说明，这是决定你是否可以完成任务的关键。
    - 【当前进度总结】 根据对话历史记录（如果存在）总和任务总结当前的状态
    - 【已收集数据】 如果任务涉及信息收集，你必须在这里详细列出目前已记录的数据
    - 【下一步计划】 你需要说明你下一步的计划/想法是什么
    - 【本轮执行动作】 1. 说明你本轮执行了什么操作。2. 为什么执行这个操作
    - 【屏幕描述】 1. 你需要描述当前的屏幕内容。2. 如果执行完当前操作下一轮屏幕内容应该发生什么样的变化。
